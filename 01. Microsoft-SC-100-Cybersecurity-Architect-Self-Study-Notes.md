# 🔐 1. Design Zero Trust Strategy & Architecture

---

## Q1. What is Zero Trust and why is it critical for modern cybersecurity?

A: Zero Trust is a security model that assumes no implicit trust is granted to assets or users based solely on location or network. Every request is authenticated, authorized, and continuously validated. It's critical because traditional perimeter-based security fails against insider threats, cloud workloads, and remote work scenarios.

---

## Q2. What are the three core principles of Zero Trust?

A:

1. Verify Explicitly – always authenticate and authorize using all available data points
2. Use Least Privilege Access – limit user access with just-in-time and just-enough-access policies
3. Assume Breach – minimize blast radius, segment access, and verify end-to-end encryption

---

## Q3. Microsoft Zero Trust framework alignment with SC-100

A: Microsoft's Zero Trust framework includes six pillars:

* Identities
* Devices
* Applications
* Data
* Infrastructure
* Networks

SC-100 requires architecture across all pillars using Azure AD, Defender, Sentinel, and Purview.

---

## Q4. Role of Conditional Access

A: Conditional Access is the Zero Trust policy engine that evaluates signals (user, device, location, risk) to enforce access decisions like MFA, compliant devices, or blocking access.

### 🔐 Q94. How do you design a data classification strategy using Microsoft Purview?

**A: Design includes:**
- Creating a Sensitive Information Type (SIT) taxonomy for business and regulatory data  
- Applying auto-labeling policies to classify data at rest and in transit  
- Deploying Purview Information Protection (formerly AIP) for Office documents  
- Configuring sensitivity labels in SharePoint, Teams, and Exchange  
- Using Content Explorer to monitor classified data distribution  

---

### 🏷️ Q95. What are Sensitivity Labels and how do they protect data?

**A:**
Sensitivity labels (configured in Microsoft Purview) apply persistent protection to documents and emails. They can:

- Encrypt content so only authorized users can access it  
- Apply visual markings (headers, footers, watermarks)  
- Control sharing (e.g., block external sharing)  
- Integrate with Conditional Access to restrict access from unmanaged devices  

---

### 🚫 Q96. How does Microsoft Purview Data Loss Prevention (DLP) work?

**A:**
DLP policies scan content across:

- Microsoft 365 (Exchange, SharePoint, Teams, OneDrive)  
- Endpoints (via Microsoft Defender for Endpoint)  
- Cloud apps (via Microsoft Defender for Cloud Apps)  

**Actions include:**
- Block sharing  
- Require business justification  
- Notify users or admins  
- Automatically encrypt content  

---
